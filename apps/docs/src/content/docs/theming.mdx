## Convention

DocyUI uses a simple <code className="text-sm bg-muted px-1 py-0.5 rounded">background</code> and <code className="text-sm bg-muted px-1 py-0.5 rounded">foreground</code> convention for colors. The <code className="text-sm bg-muted px-1 py-0.5 rounded">background</code> variable is used for the background color of a component and the <code className="text-sm bg-muted px-1 py-0.5 rounded">foreground</code> variable is used for the text color.

The <code className="text-sm bg-muted px-1 py-0.5 rounded">background</code> suffix is omitted when the variable is used for the background color of the component.

<CodeBlock 
  react={`--primary: 222.2 47.4% 11.2%;
--primary-foreground: 210 40% 98%;`}
  vue={`--primary: 222.2 47.4% 11.2%;
--primary-foreground: 210 40% 98%;`}
  language="css"
  title="CSS Variables"
/>

The <code className="text-sm bg-muted px-1 py-0.5 rounded">primary</code> variable is used for the background color of the component and the <code className="text-sm bg-muted px-1 py-0.5 rounded">primary-foreground</code> variable is used for the text color.

<CodeBlock 
  react={`<div className="bg-primary text-primary-foreground">
  Hello, world!
</div>`}
  vue={`<div class="bg-primary text-primary-foreground">
  Hello, world!
</div>`}
  title="Usage Example"
/>

## Adding New Colors

To add new colors, you need to add them to your CSS file and your <code className="text-sm bg-muted px-1 py-0.5 rounded">tailwind.config.js</code> file.

### 1. Add to CSS

<CodeBlock 
  react={`:root {
  --warning: 38 92% 50%;
  --warning-foreground: 48 96% 89%;
}

.dark {
  --warning: 48 96% 89%;
  --warning-foreground: 38 92% 50%;
}`}
  vue={`:root {
  --warning: 38 92% 50%;
  --warning-foreground: 48 96% 89%;
}

.dark {
  --warning: 48 96% 89%;
  --warning-foreground: 38 92% 50%;
}`}
  language="css"
  title="global.css"
/>

### 2. Add to Tailwind Config

<CodeBlock 
  react={`module.exports = {
  theme: {
    extend: {
      colors: {
        warning: "hsl(var(--warning))",
        "warning-foreground": "hsl(var(--warning-foreground))",
      },
    },
  },
}`}
  vue={`module.exports = {
  theme: {
    extend: {
      colors: {
        warning: "hsl(var(--warning))",
        "warning-foreground": "hsl(var(--warning-foreground))",
      },
    },
  },
}`}
  language="js"
  title="tailwind.config.js"
/>

You can now use the <code className="text-sm bg-muted px-1 py-0.5 rounded">warning</code> utility class in your components.

<CodeBlock 
  react={`<div className="bg-warning text-warning-foreground" />`}
  vue={`<div class="bg-warning text-warning-foreground" />`}
  title="Usage"
/>

## CSS Variables

DocyUI components are styled using CSS variables, which makes it easy to customize the default colors and create custom themes.

### Enabling CSS Variables

CSS variables are enabled by default when you initialize a new project. You can also enable them manually by setting <code className="text-sm bg-muted px-1 py-0.5 rounded">tailwind.cssVariables</code> to <code className="text-sm bg-muted px-1 py-0.5 rounded">true</code> in your <code className="text-sm bg-muted px-1 py-0.5 rounded">docyui.json</code> file.

<CodeBlock 
  react={`{
  "tailwind": {
    "cssVariables": true
  }
}`}
  vue={`{
  "tailwind": {
    "cssVariables": true
  }
}`}
  language="json"
  title="docyui.json"
/>

### Using CSS Variables

When CSS variables are enabled, components will use CSS variables instead of Tailwind CSS utility classes.
